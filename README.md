# ระบบขอเวร

เว็บแอปสำหรับบันทึกคำขอเวร/ลา แสดงผลบนปฏิทินแยกตามหน่วยงาน และบันทึกข้อมูลคำขอเวรลง Google Sheet ผ่าน Google Apps Script

## ไฟล์ในระบบ

- `index.html`  
  หน้าเว็บหลักของระบบ ใช้สำหรับเลือกหน่วยงาน ตำแหน่ง รายชื่อ ประเภทเวร วันที่เริ่มต้น-สิ้นสุด และดูปฏิทิน

- `google-apps-script-log.gs`  
  โค้ดสำหรับวางใน Google Apps Script เพื่อรับข้อมูลจากเว็บ บันทึกลง Google Sheet และตรวจรหัส Admin จาก Script Properties

## ความสามารถหลัก

- เลือกหน่วยงานก่อน แล้วเลือกตำแหน่งและรายชื่อจากหน่วยงานนั้น
- ขอเวรแบบมีวันเริ่มต้นและวันสิ้นสุด
- ประเภทเวร: `Vac`, `Off`, `ลากิจ`, `อบรม`, `ลาป่วย`, `ลาคลอด`, `ลาบวช`
- `ลากิจ` และ `อบรม` ต้องกรอกหมายเหตุ
- `อบรม` ห้ามขอในวันเสาร์-อาทิตย์และวันหยุดนักขัตฤกษ์
- `Vac` ห้ามขอในวันเสาร์-อาทิตย์และวันหยุดนักขัตฤกษ์
- ขอเวรติดต่อกันได้ไม่เกิน 9 วัน ยกเว้น `ลาป่วย`, `ลาคลอด`, `ลาบวช`
- โควตาต่อวันในหน่วยงานเดียวกัน ตำแหน่งเดียวกัน และ Level เดียวกัน:
  - Level 1-4 ขอซ้ำได้ไม่เกิน 2 คน
  - Level 5 ขอซ้ำได้ไม่เกิน 3 คน
- ถ้าโควตาเต็ม ระบบจะถามให้ยืนยันเป็นตัวสำรอง
- ปฏิทินแสดงรายการขอเวรตามหน่วยงานที่เลือก
- รายการหลายวันแสดงเป็นแถบลากยาวถึงวันที่สิ้นสุด
- ถ้ามีหลายรายการในวันเดียวกัน ระบบจัดเป็นเลนไม่ให้ชื่อทับกัน และแสดง `+เพิ่มเติม`
- Admin สามารถแก้ไขวันขอเวร ยกเลิกคำขอ อนุมัติตัวสำรอง และจัดการรายชื่อ/หน่วยงาน/ตำแหน่ง/Level ได้

## กฎวันที่ขอเวร

ผู้ใช้ทั่วไป:

- ต้องขอภายในวันที่ 25 ของ 2 เดือนก่อนเดือนเวร  
  ตัวอย่าง: เวรเดือนสิงหาคมต้องขอภายในวันที่ 25 มิถุนายน
- ขอเวรล่วงหน้าได้ไม่เกิน 6 เดือน

Admin:

- สามารถบันทึกหรือแก้ไขวันขอเวรได้โดยไม่จำกัดช่วงวันที่

## การใช้งานทั่วไป

1. เลือกหน่วยงาน
2. เลือกตำแหน่ง
3. เลือกรายชื่อเจ้าหน้าที่
4. เลือกวันเริ่มต้นและวันสิ้นสุด
5. เลือกประเภทเวร
6. ถ้าเป็น `ลากิจ` หรือ `อบรม` ให้กรอกหมายเหตุ
7. กดยื่นคำขอ

ถ้าโควตายังว่าง ระบบจะบันทึกสถานะ `Approved`  
ถ้าโควตาเต็ม ระบบจะถามว่าจะบันทึกเป็นตัวสำรองหรือไม่ และสถานะจะเป็น `Pending`

## การใช้งาน Admin

1. กด `สลับเป็น Admin`
2. ใส่รหัส Admin
3. ระบบจะส่งรหัสไปตรวจที่ Apps Script โดยอ่านจาก Script Properties
4. เมื่อเข้าสู่โหมด Admin แล้ว สามารถ:

- แก้ไขวันขอเวร
- ยกเลิกคำขอเวร
- อนุมัติตัวสำรอง
- จัดการรายชื่อเจ้าหน้าที่
- เพิ่ม/ลบ/แก้ไขหน่วยงาน
- เพิ่ม/ลบ/แก้ไขตำแหน่ง
- เพิ่ม/ลบ/แก้ไข Level

เมื่อลบข้อมูล ระบบจะถามยืนยันก่อนทุกครั้ง

## การแสดงผลปฏิทิน

- ปฏิทินผูกกับหน่วยงานที่เลือกด้านซ้าย
- วันเสาร์-อาทิตย์และวันหยุดนักขัตฤกษ์แสดงพื้นหลังสีเหลือง
- รายการหลายวันจะแสดงเป็นแถบลากยาวครอบวันที่ขอ
- รายการในวันเดียวกันจะจัดเป็นเลนเพื่อไม่ให้ชื่อทับกัน
- ถ้ามีรายการมากกว่าพื้นที่แสดง จะขึ้น `+เพิ่มเติม`
- คลิกวันที่หรือ `+เพิ่มเติม` เพื่อดูรายการทั้งหมดใน popup

## ข้อควรระวัง

- รหัส Admin ไม่ควรอยู่ในไฟล์โค้ด ให้เก็บใน Apps Script Script Properties เท่านั้น
- URL ของ Google Sheet ใช้เป็นปลายทางโดยตรงไม่ได้ ต้องใช้ Web App URL จาก Apps Script
- หลังแก้โค้ด Apps Script ต้อง deploy เวอร์ชันใหม่ทุกครั้ง
- ข้อมูลคำขอเวรถูกเก็บในหน่วยความจำของหน้าเว็บระหว่างใช้งาน และ log คำขอถูกส่งไป Google Sheet
- ถ้าเปิดไฟล์จากหลายเครื่องพร้อมกัน ระบบไม่ได้ sync รายการคำขอเดิมกลับจาก Google Sheet มาแสดงในปฏิทินโดยอัตโนมัติ

## การตรวจสอบเมื่อข้อมูลไม่เข้า Google Sheet

1. ตรวจว่า `GOOGLE_SHEET_LOG_URL` ใน `index.html` เป็น URL `/exec` ของ Apps Script
2. ตรวจว่า Deploy เป็น `Web app`
3. ตรวจว่า `Who has access` ตั้งเป็น `Anyone`
4. ตรวจว่า Apps Script มีสิทธิ์เข้าถึง Google Sheet
5. ถ้าเพิ่งแก้ Apps Script ให้ deploy เป็น `New version`
6. ลองยื่นคำขอใหม่ แล้วดูแถวใหม่ในแท็บ `ShiftRequests`
